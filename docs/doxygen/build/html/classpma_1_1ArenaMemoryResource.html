Constructor.

Parameters
initialSizeThe size of the first allocated region from which allocation requests are served.
regionSizeWhen a memory region backing allocation requests has not enough free space to serve an allocation, an additional region is allocated for both the current and all subsequent allocations. This parameter denotes the size of these additionally allocated regions.
growthFactorIt describes by which factor should each subsequently allocated region be scaled, relative to the previous region. A list of possible region allocation would look like:

regions = {initialSize, regionSize, regions[1] * growthFactor, regions[2] * growthFactor, ... , regions[n - 1] * growthFactor}

Parameters
upstreamThe backing memory region will be allocated using the given upstream MemoryResource.
